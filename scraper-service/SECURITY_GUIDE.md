# 安全操作指南

## 凭据保护

当前系统使用 `.env` 文件存储敏感凭据信息。为确保安全，请遵循以下最佳实践：

### 1. 文件权限
- `.env` 文件已设置为 600 权限（仅所有者可读写）
- 不要更改此权限设置
- 定期检查文件权限是否正确

### 2. 版本控制
- `.env` 文件已被添加到 `.gitignore`，不会被提交到版本控制系统
- 不要在任何公共仓库中存储真实的凭据信息

### 3. 密码管理
- 定期更换密码
- 不要在代码中硬编码密码
- 考虑使用更安全的凭据管理方案（如环境变量注入或密钥管理系统）

### 4. 系统访问控制
- 限制对服务器的物理和远程访问
- 使用强密码保护您的用户账户
- 定期审查系统访问日志

## 推荐的安全改进方案

### 方案一：环境变量注入（推荐）
不要在 `.env` 文件中存储真实密码，而是通过外部环境变量注入：

```bash
# 运行时通过环境变量传递真实凭据
SCRAPER_USERNAME="zhangx" SCRAPER_PASSWORD="Kylin39_st" python run.py --mode schedule
```

### 方案二：加密存储
使用加密技术存储密码，详情请参考项目文档中的加密方案。

## 应急响应

如果怀疑凭据已泄露：
1. 立即更改相关账户密码
2. 检查系统日志寻找异常访问
3. 通知相关人员
4. 更新 `.env` 文件中的凭据
